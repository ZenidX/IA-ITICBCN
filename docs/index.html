1. Què és el servidor d'IA?

L'Institut TIC de Barcelona disposa d'un servidor d'IA amb Ollama, accessible des de la xarxa interna del centre. Permet executar models de llenguatge grans (LLM) de forma local, sense dependre de serveis al núvol ni enviar dades fora del centre.

Què ofereix?

  • Interfície gràfica web per xatejar amb els models (Open WebUI)
  • API REST compatible amb l'estàndard OpenAI per integrar en projectes
  • Múltiples models de diferents mides i especialitzacions
  • Compatible amb n8n, LangChain, aplicacions pròpies, RAG pipelines i més

Requisit

Cal estar connectat a la xarxa interna de l'ITIC per accedir al servidor.

2. Interfície gràfica: Open WebUI

La manera més fàcil de començar és la interfície web. Simplement obre el navegador i accedeix a:

http://192.168.1.22:8080

Funcionalitats

  • Interfície intuïtiva estil ChatGPT
  • Selecció de model des del menú desplegable
  • Historial de converses
  • Gestió de models
  • Suport per a múltiples usuaris

Llistar models actualitzats

Els models poden canviar. Per veure els disponibles en temps real:

curl http://192.168.1.22:11434/api/tags

4. API Ollama

L'API REST d'Ollama permet integrar els models als vostres projectes. L'endpoint base és:

http://192.168.1.22:11434

Endpoints principals

EndpointMètodeDescripció
/api/generatePOSTGenerar text amb un model
/api/chatPOSTConversa multi-torn (amb historial)
/api/tagsGETLlistar models disponibles
/v1/chat/completionsPOSTEndpoint compatible amb OpenAI

5. Exemple amb cURL

Generació simple

curl http://192.168.1.22:11434/api/generate -d '{
  "model": "gpt-oss:120b",
  "prompt": "Explica la formulació del sulfat de coure",
  "stream": false
}'

Conversa amb historial

curl http://192.168.1.22:11434/api/chat -d '{
  "model": "qwen3.5:35b",
  "messages": [
    {"role": "user", "content": "Hola, què pots fer?"}
  ],
  "stream": false
}'

6. Exemple amb Python

Amb la llibreria requests

import requests

response = requests.post(
    'http://192.168.1.22:11434/api/generate',
    json={
        'model': 'gpt-oss:120b',
        'prompt': 'El teu prompt aquí',
        'stream': False
    }
)
print(response.json()['response'])

Instal·lació

Si no tens la llibreria requests: pip install requests

7. Compatible amb OpenAI SDK

Ollama exposa un endpoint compatible amb l'API d'OpenAI, cosa que permet reutilitzar codi existent o llibreries que ja hi funcionen.

from openai import OpenAI

client = OpenAI(
    base_url='http://192.168.1.22:11434/v1',
    api_key='ollama'  # Ollama no requereix API key real
)

response = client.chat.completions.create(
    model='gpt-oss:120b',
    messages=[{'role': 'user', 'content': 'Hola!'}]
)
print(response.choices[0].message.content)

Instal·lació

Cal tenir l'SDK: pip install openai

1. Què és OpenCode?

OpenCode és una eina CLI (Command Line Interface) de codi obert per a assistència de programació amb IA. Funciona de manera similar a altres assistents de codi, però amb l'avantatge de poder connectar-se a models locals via Ollama o a diversos proveïdors cloud.

Característiques principals

  • Interfície de terminal interactiva
  • Suport per a múltiples proveïdors d'IA (Ollama, OpenAI, Anthropic, etc.)
  • Capacitats agèntiques (lectura/escriptura de fitxers, execució de comandes)
  • Integració amb MCP (Model Context Protocol)
  • Sessions persistents i exportables

2. Requisits previs

  • Sistema operatiu: Windows 10/11, macOS o Linux
  • Node.js: versió 18 o superior
  • Git (recomanat)
  • Connexió a la xarxa interna de l'ITIC

Verificar Node.js

node --version
# Ha de mostrar v18.x.x o superior

5. Configuració

Pas 1: Crear el directori

New-Item -ItemType Directory -Force -Path "$env:USERPROFILE\.config\opencode"
mkdir -p ~/.config/opencode

Pas 2: Crear el fitxer de configuració

Ruta del fitxer:

  • Windows: %USERPROFILE%\.config\opencode\opencode.json
  • macOS/Linux: ~/.config/opencode/opencode.json

Contingut del fitxer:

{
  "$schema": "https://opencode.ai/config.json",
  "provider": {
    "ollama-itic": {
      "npm": "@ai-sdk/openai-compatible",
      "name": "Ollama ITIC Barcelona",
      "options": {
        "baseURL": "http://192.168.1.22:11434/v1"
      },
      "models": {
        "gpt-oss:120b": {
          "name": "GPT-OSS 120B (Principal)",
          "tools": true
        },
        "qwen3.5:35b": {
          "name": "Qwen 3.5 35B (Rapid)",
          "tools": true
        },
        "danielsheep/Qwen3-Coder-30B-A3B-Instruct-1M-Unsloth:UD-Q6_K_XL": {
          "name": "Qwen3 Coder 30B",
          "tools": true
        },
        "gemma3:270m": {
          "name": "Gemma 3 270M (Test)",
          "tools": true
        }
      }
    }
  }
}

7. Utilitzar OpenCode

Iniciar amb un model concret

# Model principal (120B) - màxima qualitat
opencode -m ollama-itic/gpt-oss:120b

# Model ràpid (35B) - bon equilibri
opencode -m ollama-itic/qwen3.5:35b

# Model de codi (30B) - especialitzat en programació
opencode -m "ollama-itic/danielsheep/Qwen3-Coder-30B-A3B-Instruct-1M-Unsloth:UD-Q6_K_XL"

# Model lleuger (270M) - proves ràpides
opencode -m ollama-itic/gemma3:270m

Altres opcions

# Iniciar en un projecte específic
opencode /ruta/al/teu/projecte

# Continuar l'última sessió
opencode --continue

Exemples de peticions

  • "Analitza l'estructura d'aquest projecte"
  • "Crea una funció per validar correus electrònics"
  • "Explica què fa el fitxer main.py"
  • "Afegeix tests unitaris per a utils.py"

9. Resolució de problemes

Error: "Connection refused"

  • Verifica que estàs connectat a la xarxa interna de l'ITIC
  • Comprova que el servidor està funcionant: curl http://192.168.1.22:11434

Error: "Model not found"

  • Llista els models disponibles: curl http://192.168.1.22:11434/api/tags
  • Verifica que el nom del model és correcte a la configuració

Les eines (tools) no funcionen

La finestra de context del model pot ser massa petita. Si tens accés al servidor:

ollama run qwen3.5:35b
>>> /set parameter num_ctx 32768
>>> /save qwen3.5:35b-32k
>>> /bye

OpenCode va lent

  • Utilitza un model més petit per a proves (gemma3:270m)
  • El model de 120B requereix més temps de processament
